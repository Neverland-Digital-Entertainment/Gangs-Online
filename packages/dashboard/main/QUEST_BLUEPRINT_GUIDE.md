# 任務藍圖系統 - 節點說明書
## Quest Blueprint System - Node Reference Guide

---

## 概述 (Overview)

任務藍圖系統使用 **React Flow** 視覺化編輯器，讓設計師透過拖拉節點和連線來建立完整的任務流程。每個任務藍圖儲存於 Firebase `quest_blueprints` 集合中。

### 操作方式
- **新增節點**：點擊畫布上方的節點按鈕（如 `+ 開始`、`+ 對話`）
- **連線**：從節點底部的圓點拖拉到另一個節點頂部的圓點
- **移動節點**：直接拖拉節點
- **刪除**：選取節點或連線後按 `Delete` 或 `Backspace`
- **縮放**：滾輪縮放，左下角工具列也有縮放按鈕
- **小地圖**：右下角顯示整體藍圖的縮略圖

## 節點說明

### 1. 開始節點 (Start Node) 🟢

| 顏色 | 綠色 |
|------|------|
| 輸入端 | 無 |
| 輸出端 | 1 個（底部） |

每個任務藍圖**必須有且僅有一個**開始節點，作為任務的觸發入口。

| 欄位 | 說明 | 範例 |
|------|------|------|
| NPC 模板 ID | 觸發此任務的 NPC 模板 ID（來自 `npc_templates` 集合） | `quest_giver_01` |
| 最低等級 | 玩家需達到的最低等級才能接取此任務（選填） | `5` |
| 最高等級 | 超過此等級將無法接取（選填） | `20` |
| 前置任務 | 需先完成的任務藍圖 ID，用於實現連環任務 | `q_first_blood` |

**設計提示**：
- 連環任務：在任務 B 的開始節點填入任務 A 的 ID 作為前置任務
- 玩家必須與指定 NPC 對話才能觸發任務（手動接取節奏）

---

### 2. 對話節點 (Dialogue Node) 🔵

| 顏色 | 藍色 |
|------|------|
| 輸入端 | 1 個（頂部） |
| 輸出端 | 1 個（底部） |

用於呈現 NPC 對話內容，支援雙語文本。可串聯多個對話節點實現多段對話。

| 欄位 | 說明 | 範例 |
|------|------|------|
| 說話者 ID | 說話的 NPC 模板 ID | `npc_chen` |
| 表情 | 表情代碼，影響客戶端演出（選填） | `happy`、`angry`、`sad` |
| 文本（繁中） | 繁體中文對話內容 | `你好，年輕人！我需要你的幫忙。` |
| 文本（英文） | 英文對話內容 | `Hello, young one! I need your help.` |

**設計提示**：
- 連續對話：將多個對話節點依序連線
- 不同角色對話：更換 `說話者 ID` 即可
- 表情代碼需與客戶端支援的表情系統對應

---

### 3. 選擇節點 (Choice Node) 🟣

| 顏色 | 紫色 |
|------|------|
| 輸入端 | 1 個（頂部） |
| 輸出端 | 每個選項右側各 1 個 |

讓玩家從多個選項中做選擇，每個選項連接到不同的後續流程。

| 欄位 | 說明 | 範例 |
|------|------|------|
| 選項文字（繁中） | 選項按鈕的繁中文字 | `我願意幫忙` |
| 選項文字（英文） | 選項按鈕的英文文字 | `I'll help you` |

**操作**：
- 點擊「+ 新增選項」按鈕增加選項
- 每個選項右側會出現一個輸出端口，將其連接到對應的後續節點
- 點擊 ✕ 可刪除選項

**設計提示**：
- 至少需要 2 個選項才有分支意義
- 可用於接受/拒絕任務、劇情分歧等場景

---

### 4. 任務目標節點 (Task Node) 🟠

| 顏色 | 橘色 |
|------|------|
| 輸入端 | 1 個（頂部） |
| 輸出端 | 1 個（底部）- 完成後觸發 |

定義玩家需要完成的具體目標。流程會暫停在此節點，直到玩家完成目標。

| 目標類型 | 說明 | 需要填寫的欄位 |
|----------|------|---------------|
| **收集道具** (collect) | 收集指定道具 | 目標 ID = 道具 ID，所需數量 |
| **擊殺敵人** (kill) | 擊殺指定敵人 | 目標 ID = 敵人模板 ID，所需數量 |
| **與 NPC 對話** (interact) | 與指定 NPC 交互 | 目標 ID = NPC 模板 ID，所需數量（通常為 1） |
| **抵達位置** (location) | 限時內抵達指定區域 | X 座標、Z 座標、半徑 |

**欄位說明**：

| 欄位 | 說明 | 範例 |
|------|------|------|
| 目標類型 | 下拉選擇上述四種類型之一 | `kill` |
| 目標 ID | 對應的道具/敵人/NPC 的 ID | `enemy_thug_01` |
| 所需數量 | 需要完成的次數 | `5` |
| X / Z / 半徑 | 僅 `location` 類型需要，定義目標區域 | `100, -200, 10` |

---

### 5. 條件判定節點 (Condition Node) 🟡

| 顏色 | 黃色 |
|------|------|
| 輸入端 | 1 個（頂部） |
| 輸出端 | 2 個（底部左=成功，底部右=失敗） |

即時檢查玩家的狀態，根據結果走不同分支。**不會暫停流程**，是瞬間判定。

| 條件類型 | 說明 | 範例 |
|----------|------|------|
| **金錢檢查** (money) | 檢查玩家金錢是否 ≥ 所需數量 | 所需數量 = `500` |
| **道具檢查** (item) | 檢查玩家是否持有足夠數量的指定道具 | 目標 ID = `key_item_01`，所需數量 = `1` |
| **變數檢查** (variable) | 檢查全域變數的值 | 目標 ID = `has_met_boss`，所需數量 = `1` |

**設計提示**：
- 綠色輸出端（左側）= 條件滿足時走的路線
- 紅色輸出端（右側）= 條件不滿足時走的路線
- 常見用法：檢查玩家是否有足夠金錢/道具才能推進劇情

---

### 6. 動作執行節點 (Action Node) 🔴

| 顏色 | 紅色 |
|------|------|
| 輸入端 | 1 個（頂部） |
| 輸出端 | 1 個（底部） |

執行一次性的系統操作，**瞬間觸發不暫停**。

| 動作類型 | 說明 | 需要的欄位 |
|----------|------|-----------|
| **扣除道具** (remove_item) | 從玩家背包移除指定道具 | 目標 ID = 道具 ID，數量 |
| **扣除金錢** (remove_money) | 扣除玩家金錢 | 數量 |
| **生成 NPC** (spawn_npc) | 在客戶端實例化一個臨時 NPC（Phasing） | 目標 ID = NPC 模板 ID |
| **設定變數** (set_variable) | 設定全域變數的值 | 目標 ID = 變數名稱，值 |

**設計提示**：
- 搭配條件節點使用：先檢查是否有足夠道具，再扣除
- 生成 NPC 用於任務中途出現的臨時角色（僅在客戶端渲染）
- 設定變數可用於跨任務的全域狀態追蹤

---

### 7. 結束節點 (End Node) ⚫

| 顏色 | 灰色 |
|------|------|
| 輸入端 | 1 個（頂部） |
| 輸出端 | 無 |

任務流程的終點，定義任務完成後發放的獎勵。

| 欄位 | 說明 | 範例 |
|------|------|------|
| 獎勵經驗值 | 完成任務獲得的 XP | `500` |
| 獎勵 HKD | 完成任務獲得的遊戲貨幣 | `1000` |
| 獎勵道具 | 完成任務獲得的道具列表 | 道具 ID = `sword_01`，數量 = `1` |

**操作**：
- 點擊「+ 新增獎勵道具」添加道具獎勵
- 每行填入道具 ID 和數量
- 點擊 ✕ 可移除道具獎勵

**設計提示**：
- 一個任務可以有多個結束節點（用於不同分支的不同獎勵）
- 到達結束節點後，伺服器會更新玩家的 `Completed_Quests` 旗標

## 典型任務流程範例

### 範例 1：簡單擊殺任務
```
[開始] → [對話: 任務介紹] → [目標: 擊殺 5 個混混] → [對話: 完成感謝] → [結束: 獎勵]
```

### 範例 2：帶選擇的劇情任務
```
[開始] → [對話: NPC 求助] → [選擇: 幫忙/拒絕]
                                  ├─ 幫忙 → [目標: 收集 3 個道具] → [對話: 感謝] → [結束: 豐厚獎勵]
                                  └─ 拒絕 → [對話: 失望] → [結束: 少量獎勵]
```

### 範例 3：條件分支任務
```
[開始] → [對話: 需要通行費] → [條件: 金錢 ≥ 500]
                                  ├─ 成功 → [動作: 扣除 500 金] → [對話: 通行] → [結束]
                                  └─ 失敗 → [對話: 金錢不足] → [目標: 擊殺獲取] → [結束]
```

### 範例 4：連環任務（跨藍圖）
```
任務 A 藍圖: [開始: 無前置] → ... → [結束]
任務 B 藍圖: [開始: 前置=任務A ID] → ... → [結束]
```
> 玩家完成任務 A 後，需再次與 NPC 對話才能觸發任務 B

## 任務狀態機

每位玩家對應各任務有以下四種狀態：

| 狀態 | 說明 | NPC 提示 |
|------|------|---------|
| **Locked** | 未滿足前置任務或等級不足 | 無提示 |
| **Available** | 可接取 | 驚嘆號 ❗ |
| **Active** | 任務進行中 | 任務日誌顯示目標 |
| **Completed** | 已完成，獎勵已發放 | 存入歷史列表 |

## Firebase 資料結構

```
quest_blueprints/
  {blueprintId}/
    name: string            // 任務名稱
    description: string     // 任務描述
    nodes: []               // React Flow 節點陣列
    edges: []               // React Flow 連線陣列
    isActive: boolean       // 是否啟用
    createdAt: Timestamp
    updatedAt: Timestamp
```
